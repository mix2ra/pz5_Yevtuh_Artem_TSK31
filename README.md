**Відповіді на контрольні питання**

---
№1. Порівняйте та назвіть переваги та недоліки таких методів сортування: сортування бульбашкою, сортування вибором, сортування вставкою.

Сортування бульбашкою:
Переваги — один із найпростіших для розуміння алгоритмів, добре працює на малих наборах даних, а при майже впорядкованому масиві може завершити роботу завчасно.
Недоліки — повільний метод для великих масивів, виконує багато зайвих перестановок, має квадратичну часову складність.

Сортування вибором:
Переваги — елементарна логіка реалізації та мінімальна кількість реальних обмінів місцями.
Недоліки — продуктивність низька при великих обсягах даних, складність завжди залишається O(n²), та алгоритм не забезпечує стабільності.

Сортування вставкою:
Переваги — чудово справляється з масивами, де значна частина елементів уже впорядкована; стабільний; реалізація нескладна.
Недоліки — працює повільно, якщо дані дуже перемішані; у гіршому випадку також має складність O(n²).

---

№2. Оцініть кожен з методів відповідно до наступних критеріїв:
час роботи;
потреби у додатковій пам’яті;
стабільність.
За якими ще критеріями ви можете порівняти ці алгоритми?

За швидкодією усі три методи зазвичай виконуються за O(n²), але алгоритм вставки здатен працювати значно швидше на впорядкованих або частково впорядкованих даних.
Усі вони використовують сталий обсяг додаткової пам’яті, тобто не вимагають додаткових структур — O(1).
Щодо стабільності: бульбашкове та вставкою зберігають порядок однакових елементів, тоді як вибором цього не гарантує.
Також ці алгоритми варто порівнювати за кількістю виконаних операцій, чутливістю до початкового стану даних та складністю програмної реалізації.

---

№3. Який випадок є найкращім або найгіршим для роботи цих алгоритмів? (Наприклад: частково відсортований масив або масив, відсортований у зворотному порядку). Який з методів забезпечить у цих випадках найкращій/найгірший результат?

Для бульбашкового сортування найвигідніша ситуація — повністю впорядкований масив, бо алгоритм швидко виявляє, що робота майже завершена. Найгірший випадок — повністю зворотний порядок елементів.
У сортування вибором найліпшого чи найгіршого варіанта фактично немає — кількість порівнянь не змінюється залежно від структури даних.
Сортування вставкою найшвидше працює з масивами, де елементи розташовані майже у вірному порядку, а найдовше — якщо масив відсортований у зворотному напрямку.

---

№4. Що означають позначення O(1), O(n), O(n²)?

Ці нотації описують, як швидкість роботи алгоритму залежить від кількості даних.
O(1) — час виконання залишається сталим незалежно від розміру масиву.
O(n) — тривалість роботи збільшується лінійно разом із ростом кількості елементів.
O(n²) — час зростає пропорційно квадрату кількості елементів, тобто збільшення масиву вдвічі приблизно збільшує час роботи в чотири рази.
